Offline setup guide

ChatGPT/Codex 代理启动器

让桌面版 ChatGPT/Codex 使用你指定的本地代理，不开启 Windows 系统代理，不需要 TUN，也不运行额外后台服务。

本页完全离线配置不会上传不修改系统代理启动后立即退出
1
准备代理确认软件和本地端口
2
保存配置默认配置也可直接使用
3
双击启动关闭已有 App 后运行
1

先确认你的情况

多数 V2RayN 用户不需要修改任何配置。

使用 V2RayN，mixed 端口是 10808？

发布包里的 config.json 已经是这个出厂配置。启动 V2RayN 后，可以直接进入第 3 步。

01
你已经有可用的代理软件和节点
本项目不提供节点，也不会替你启动 V2RayN、Clash 或其他代理软件。
02
你使用的是 Windows x64 商店版 ChatGPT
当前不支持 macOS、网页版 ChatGPT 或任意其他程序。
03
运行启动器前会完全退出已有 ChatGPT/Codex
Chromium 代理参数通常只对第一个 App 进程生效。
2

确认本地代理配置

预设只负责填入常见值，不会检测或控制你的代理软件。

如果你在代理软件中改过端口，请选择最接近的预设后再修改下面的端口。

mixed 端口通常同时支持 HTTP 与 SOCKS5，本项目默认通过 HTTP CONNECT 连接。

127.0.0.1 表示“这台电脑”。普通用户通常不需要修改。

填写代理软件界面的本地/mixed 端口，不是节点服务器端口或远程 443。

端口不确定？ 打开代理软件设置，寻找“mixed port”“混合端口”“HTTP 端口”或“SOCKS 端口”。
3

保存配置并启动

config.json 必须与 CodexProxyLauncher.exe 位于同一个文件夹。

当前配置有效。保存时请选择 CodexProxyLauncher.exe 所在文件夹。
配置已生成，接下来：
  1. 确认代理软件正在运行且节点可用。
  2. 完全退出已有 ChatGPT/Codex，包括后台进程。
  3. 回到此文件夹，双击 CodexProxyLauncher.exe。
高级设置：日志、QUIC 与本地直连列表

诊断日志 出厂：仅错误时记录

仅错误（推荐）：正常启动不创建日志目录，出错时才写入本次启动上下文。
始终记录：从启动开始持续写入，适合反复排错。
关闭：完全不写日志，错误弹窗会提示日志已关闭。

日志可能包含启动器与 App 路径、代理地址和错误信息；不会记录账号、对话内容或代理流量。分享日志前仍请自行检查。

禁用 QUIC 出厂：开启，建议保持

最终添加 --disable-quic，让 App 优先使用可由本地 HTTP/SOCKS5 代理承载的 TCP，减少 UDP 连接绕过代理或直接失败的可能。

本地与内网直连列表 出厂：回环地址和常见私网

这些目标不会交给代理，可避免本地服务失效或出现代理循环。不了解 Chromium bypass 语法时请保留默认值。

专家工具：JSON、等价命令与导入

        遇到问题：按现象排查
        

          
提示“无法连接本地代理”
确认代理软件正在运行；再核对当前端口是否等于软件界面显示的本地/mixed 端口。

          
提示“ChatGPT/Codex 已经运行”
从系统托盘和任务管理器完全退出桌面 App，然后重新双击启动器。终端里的 Codex CLI 不受影响。

          
配置保存后没有变化
检查新 config.json 是否真的覆盖到了启动器旁边，而不是仍留在浏览器“下载”文件夹。

          
仍然无法解决
默认仅在启动出错时生成同目录 logs 文件夹。若需要完整启动轨迹，可在高级设置中临时选择“始终记录”；若已关闭日志，错误弹窗会明确提示。
